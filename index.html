Stage 1.3: Adding CSS Style and HTML Strucutre

1. So what is CSS?


A. Cascading Style Sheets (CSS) can serve as a search and replacement tool.
B. "Cascading" means that rules apply not only to the elements, but also to all of the those elements child emelements. However, if a child element has multiple, overlapping rules defined for it, the more special rules takes effect.
C. By identifying the class, you can then avoid repition in the coding by setting different properties and values.
D. Order matters when you define the rules and in what order they are applied, as precedence takes effect for styling rules.

2. Inheritance


A. Inheritance is a key feature in a CSS Ancestor-Descendant relationship.
B. It serves as a mechanism by which properties are applied not only to a specified element, but also its descendants.
C. Descendant elements may inherit CSS property values from an ancenstor element enclosing them.
D. In general, descendant elements inherit text related properties, but box related properties are not inherited.
E. Inheritance prevents certain properties from being declared over & over in a style sheet by: enhancing faster-loading of web pages by users, and enabling clients to save $$$ on bandwidth & development costs.

3. Repitition


A. Repitition is inevitable when coding, and repitition leads to errors.
B. By utilizing the class system to apply styles to sections of the code, errors can be avoided.
C. Avoiding repitition helps others read your code if any review is necessary.
